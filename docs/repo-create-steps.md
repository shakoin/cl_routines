# Repository Creation Notes

This public repo is intended to be a clean template.

## Recommended setup

1. Keep this repo public and generic.
2. Keep private routine context elsewhere.
3. Do not commit generated reports.
4. Do not commit real runtime state.
5. Use `config.example.md` as a guide for private configuration.

## If creating a private fork

A private fork or private copy can include your real operational context, such as:

- specific services;
- exact hardware;
- model baseline;
- thresholds;
- private report destination;
- custom source lists.

Do not merge private customization back into the public template.

## Suggested routine names

Generic public names:

- Daily Tech Scout
- Weekly Tech Deep Sweep

Private names can be personalized in Claude Routine UI, but do not need to be committed here.
